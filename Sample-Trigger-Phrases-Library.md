## üìä Trigger Phrase Best Practices

### Do's
‚úÖ **Include variations**: People ask the same question many ways
‚úÖ **Use natural language**: How people actually talk, not formal language
‚úÖ **Add common misspellings**: Users make typos
‚úÖ **Include acronyms**: LMS, PTO, OOO, etc.
‚úÖ **Use question formats**: "How do I...", "What is...", "Where can I..."
‚úÖ **Add urgency phrases**: "urgent", "asap", "now", "immediately"
‚úÖ **Include role-specific terms**: Job titles, departments, levels

### Don'ts
‚ùå **Too specific**: Trigger phrases should be broad enough
‚ùå **Overlapping phrases**: Don't use same phrase in multiple topics
‚ùå **Too technical**: Include plain language versions
‚ùå **Only formal language**: Include casual variations
‚ùå **Ignore synonyms**: "help", "support", "assistance" all mean the same

### Testing Your Trigger Phrases
1. Ask colleagues how they would ask the question
2. Review chatbot analytics for unrecognized phrases
3. Monitor support ticket language
4. Test with different phrasings
5. Add new phrases based on actual user input
